Part 1b Readme

In this part of the homework, all you really have to do is run the X-version of the program a couple of times with different parameters until you find one that results in a good mapping. Sample session:

  1. Log in on a Linux machine, if necessary, and set your display as in part Ia.
  2. For this part, the XENVIRONMENT should be set to point to the resources file in the part1b directory instead: export XENVIRONMENT=/u/www/users/risto/cs394n/private/hw1/part1b/XFmrevised OR from the hw1/part1b directory on your machine export XENVIRONMENT=`pwd`/XFmrevised
  3. Prepare your simulation file using 2dsimu as the starting point:
    • Remove everything below the snapshot definitions (as in part1a).
    • Change the network sizes (the nx and ny parameters), the learning rates and neighborhoods (see Readme.parameters for details) as necessary to get a good mapping.
    • Change the tend parameter to however many epochs you want to train. At first it is a good idea to run it only for a few hundred epochs to get an idea how it is going to organize; after you are confident that the parameters are correct, you can run several thousand epochs in the background (see part1a Readme for details).
    • Change the snapshot definition so that it will match tend (so that the last snapshot will be saved).
  4. Run xsofm. The same set of windows as in the previous part should pop up: input, network activity, network structure, and a command window, although this time the network activity window should be square since the network is two-dimensional.
  5. In the command window, press FILE, specify your simulation file, press RESTART, sit back, and watch the show.
  6. If the network does not look good, repeat steps 3-5 with different parameters until you are happy with it. Note that currently the largest netsize is 1000x1000; if you want to try larger nets, you have to change maxnx and maxny in sofmdefs.h and recompile.

For more details, see the More details section of the README for part 1a, noting that the 2dsimu file should be used throughout instead of the 1dsimu file.
