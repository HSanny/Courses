Part 3a Readme

This part of the homework does not require any training, just running the program in this directory with your data to check how well the feature map you trained is performing in the classification task.

  1. Copy your simulation file (e.g., mysimu) from part 2. If you saved multiple snapshots, edit the copy so that it only contains the last snapshot (See part2/Readme.parameters for details on the structure of the snapshot). The file simu.demo is an example of what it should look like.
  2. Copy your training vector and label files (e.g. train.vectors and train.labels) from part 2.
  3. Edit sofm.tcl so that it will read mysimu (or whatever your simulation file) instead of simu.demo.
  4. Type ./sofm.tcl. A feature map window will pop up, as in Part 2. Click on the STEP button once.
    • the centermost label in each box identifies the closest training vector for that unit (i.e. the label of that vector in the training vectorfile that is closest in Euclidean distance to the weight vector of the unit).
    • the rest of the labels indicate where the testvectors were mapped.
    • classification performance is shown on the bottom left of the window

    If a testvector label is not the same as the label of the unit (i.e. the centermost label in the box), that means the network classified it incorrectly.
